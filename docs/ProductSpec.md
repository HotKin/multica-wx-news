# 主流新闻汇集微信小程序 — Product Spec

---

## 背景

用户需要一个轻量级的新闻获取入口，能够在微信生态内快速浏览主流新闻，无需下载独立 App。微信小程序天然具备免安装、即用即走的优势，适合碎片化的阅读场景。

当前项目基础框架已初始化完成，包含首页、详情页及 mock 数据。本 Spec 旨在明确产品需求边界和业务规则，为后续技术迭代提供可验收的标准。

---

## 目标用户

| 用户画像 | 特征 | 核心诉求 |
|---------|------|---------|
| 通勤族 | 地铁/公交上的碎片时间 | 快速浏览头条，了解大事 |
| 信息敏感型用户 | 关注多领域动态 | 按分类精准获取感兴趣的新闻 |
| 轻度阅读者 | 不安装新闻类 App | 在微信内一站式获取资讯 |

**用户共性**：追求信息获取效率，偏好简洁直观的交互，对加载速度和内容呈现有较高要求。

---

## 产品目标

1. **信息聚合**：汇集主流新闻来源，覆盖多领域（科技、国际、财经、国内、体育、教育等），让用户在一个小程序内获取全面资讯。
2. **高效浏览**：支持分类筛选、下拉刷新、上拉加载，降低用户获取信息的时间成本。
3. **深度阅读**：新闻详情页提供完整内容、图片预览，支持分享传播。
4. **体验流畅**：保证页面加载速度、滚动流畅度，符合微信小程序原生体验标准。

---

## 非目标（本次不做）

| 非目标项 | 说明 | 备注 |
|---------|------|------|
| 用户登录/账号体系 | 不区分用户身份，无需注册登录 | 后续版本可考虑 |
| 个性化推荐算法 | 不基于用户行为做 AI 推荐 | 按分类 + 时间排序 |
| 评论/点赞/收藏 | 不开放用户互动功能 | 后续版本可考虑 |
| 夜间模式/字体大小调节 | 不开放阅读偏好设置 | 后续版本可考虑 |
| 视频新闻 | 仅支持图文新闻，不支持视频播放 | 后续版本可考虑 |
| 实时推送/订阅 | 不推送消息通知 | 小程序限制，暂不考虑 |

## 使用场景

### 场景 1：通勤速览（首页浏览）
> 小张早上坐地铁上班，打开小程序，首页展示推荐新闻，他滑动浏览标题和摘要，快速了解今天的大事。

### 场景 2：分类聚焦（分类切换）
> 小李对科技新闻特别感兴趣，点击首页顶部的"科技"标签，列表刷新为仅展示科技类新闻。

### 场景 3：深度阅读（详情页）
> 小王看到一条感兴趣的财经新闻，点击进入详情页，阅读完整正文，点击文章中的配图可放大预览。

### 场景 4：社交分享
> 小陈读到一条有价值的教育新闻，点击分享按钮，转发到微信群或朋友圈，供朋友阅读。

### 场景 5：持续刷新（下拉/上拉）
> 小赵下拉首页刷新获取最新新闻，滑到底部自动加载更多历史新闻。

## 业务规则

### 新闻数据规则
1. **分类体系**：新闻分为 6 个固定分类：科技、国际、财经、国内、体育、教育。
2. **每条新闻字段**：标题（必填，≤50 字）、摘要（必填，≤100 字）、正文（必填）、封面图（必填，建议比例 16:9）、分类（必填，单选）、来源（必填）、发布时间（必填）。
3. **排序规则**：默认按发布时间倒序排列，最新发布的在前。
4. **轮播图规则**：展示 3-5 条最新或编辑精选的头条新闻，支持自动轮播（间隔 5 秒），用户手动滑动可中断自动轮播。

### 列表展示规则
5. **分页规则**：首页列表每页加载 10 条新闻，上拉到底部触发加载下一页，无更多数据时显示"没有更多了"提示。
6. **下拉刷新规则**：下拉距离超过阈值触发刷新，重新拉取最新数据，刷新完成后回到列表顶部。
7. **分类切换规则**：切换分类标签时，列表重置为第一页，按新分类重新加载。
8. **空状态规则**：某分类下无新闻时，展示空状态提示"暂无新闻，去看看其他分类吧"。

### 详情页规则
9. **图片预览**：详情页内所有图片支持点击放大预览，多张图片支持左右滑动切换，双指缩放。
10. **分享规则**：调用微信小程序原生分享能力，分享卡片显示新闻标题 + 封面图，点击卡片进入该新闻详情页。
11. **返回规则**：详情页左上角显示返回按钮，点击返回上一页（首页或列表页）。

## 页面/交互要求

### 首页

**布局结构**（从上到下）：
1. **顶部导航栏**：小程序原生导航栏，显示小程序名称"主流新闻"。
2. **分类标签栏**：横向滚动标签栏，包含"全部"+ 6 个分类标签，当前选中标签高亮显示，标签切换时列表平滑过渡。
3. **轮播图区域**：位于分类标签下方，占满屏幕宽度，高度约 200rpx，底部显示指示器（小圆点），显示当前新闻标题（底部遮罩层）。
4. **新闻列表区域**：
   - 每条新闻为卡片式布局，包含左侧封面图（正方形，约 180rpx）+ 右侧信息区（标题、摘要、来源+时间）。
   - 标题最多 2 行，超出省略；摘要最多 2 行，超出省略。
   - 列表项之间有一定间距（约 20rpx），背景色区分。
5. **底部加载提示**：加载中显示"加载中..."，无更多数据显示"—— 没有更多了 ——"。

**交互行为**：
- 下拉：手指下拉超过 60rpx 触发刷新，显示刷新动画，完成后列表回到顶部并更新数据。
- 上拉：列表滚动到底部（距底部 100rpx 内）触发加载更多，显示加载动画，追加数据到列表底部。
- 点击：点击列表项进入详情页；点击轮播图直接进入对应新闻详情页。

### 详情页

**布局结构**（从上到下）：
1. **顶部导航栏**：显示返回按钮 + 新闻标题（标题过长时截断）。
2. **新闻标题区**：大字号标题（约 40rpx，加粗），下方显示来源和发布时间（小字号，灰色）。
3. **正文内容区**：常规正文字号（约 30rpx），行高 1.6 倍，段落之间留空。
4. **图片区**：正文中的图片按原比例展示，宽度撑满内容区，图片之间留空。

**交互行为**：
- 返回：点击左上角返回按钮或系统返回手势，返回上一页。
- 图片预览：点击图片进入全屏预览模式，支持左右滑动、双指缩放，点击任意位置退出预览。
- 分享：点击右上角"..."菜单中的"转发"，或页面内分享按钮，唤起微信分享面板。

## 异常情况

| 异常场景 | 表现 | 处理策略 |
|---------|------|---------|
| 网络断开/请求失败 | 列表加载失败 | 展示"网络开小差了，点击重试"提示，提供重试按钮 |
| 下拉刷新超时 | 刷新动画持续超过 10 秒 | 自动结束刷新动画，提示"刷新失败，请检查网络" |
| 上拉加载失败 | 加载更多失败 | 保留已有数据，底部提示"加载失败，上拉重试" |
| 详情页数据加载失败 | 页面空白或报错 | 展示"内容加载失败"，提供返回首页和重试按钮 |
| 图片加载失败 | 封面图或正文图显示裂图 | 显示默认占位图（灰色背景 + 图片图标） |
| 分享失败 | 用户取消分享或微信接口报错 | 静默处理，不做强提示 |
| 某分类无新闻 | 列表为空 | 展示空状态插画 + 文案"暂无新闻，去看看其他分类吧" |

## 验收标准

### 首页验收

- [ ] 打开小程序，首页默认展示"全部"分类的新闻列表，列表按时间倒序排列。
- [ ] 顶部轮播图自动轮播，间隔 5 秒，手动滑动后暂停自动轮播，5 秒后恢复。
- [ ] 横向滑动分类标签可切换分类，切换后列表数据即时更新（有加载动画）。
- [ ] 下拉首页可触发刷新，刷新完成后列表回到顶部，数据更新。
- [ ] 上滑到列表底部触发加载更多，新数据追加到列表末尾。
- [ ] 列表为空时显示空状态提示，而非空白页面。
- [ ] 点击列表项正常跳转到详情页，点击轮播图跳转到对应新闻详情页。

### 详情页验收

- [ ] 详情页正确展示新闻标题、来源、发布时间、正文内容。
- [ ] 正文中的图片正常显示，点击图片可全屏预览，支持左右滑动和双指缩放。
- [ ] 点击返回按钮可回到首页，且首页保持之前的滚动位置和分类状态。
- [ ] 分享功能正常，分享卡片显示新闻标题和封面图，好友点击卡片进入详情页。

### 数据验收

- [ ] 至少包含 8 条模拟新闻数据，覆盖科技、国际、财经、国内、体育、教育 6 个分类。
- [ ] 每条新闻包含完整的标题、摘要、正文、封面图、分类、来源、发布时间字段。

### 体验验收

- [ ] 页面首屏加载时间 ≤ 2 秒（模拟数据场景下）。
- [ ] 列表滚动流畅，无卡顿、白屏。
- [ ] 各页面适配不同屏幕尺寸（iPhone SE ~ iPhone 15 Pro Max 等常见机型）。
