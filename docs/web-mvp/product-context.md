# 快闻 Web MVP 产品上下文

## 背景

快闻 Web 当前定位为“主流新闻聚合与展示平台”，首期以响应式 Web 验证内容消费链路、信息架构、热榜/专题/搜索价值和基础运营闭环。

本文件基于 KIN-15 产品 SDD 与 KIN-20 产品澄清同步，记录进入 OpenSpec 和实现阶段的产品边界。

## 用户

| 用户 | 目标 |
| --- | --- |
| 匿名读者 | 浏览首页、频道、详情、热榜、专题、搜索和分享内容。 |
| 注册用户 | 后续通过账号体系使用收藏接口；当前微信登录入口先不实现。 |
| 内容编辑 | 维护草稿、来源内容、频道归属并提交审核。 |
| 审核人员 | 处理自动审核结果、人工复核、投诉审核、下架和恢复。 |
| 内容运营 | 管理来源、热榜、专题、定时发布、批量发布和指标数据。 |
| 管理员 | 管理账号权限、审核规则、来源配置、日志与审计。 |

## MVP 范围

- 公开 Web：首页、频道列表、新闻详情、热榜、专题、搜索、分享、SEO、响应式适配。
- 内容后台：来源管理、文章管理、频道管理、推荐位、热榜、专题、审核、发布、下架、基础访问数据。
- 内容流程：第三方抓取入库、标准化/去重、自动审核、人工审核、系统发布、投诉下架、恢复。
- 热榜：同步第三方热榜快照；前台只展示已发布且未下架内容/专题。
- 专题：支持人工创建专题，也支持系统按热榜/相似新闻生成待审核专题。
- 收藏：保留服务端接口边界；当前微信登录入口先不实现时，前台不强行暴露账号入口。
- 埋点：阅读、登录、收藏、搜索、分享、热榜、专题、审核、发布、投诉和后台操作事件。

## 非目标

- 不做评论、私信、UGC 投稿、会员、付费阅读、复杂个性化推荐。
- 不做 QQ、Apple、Google 等第三方登录；首批曾收敛为微信，但最新口径是微信登录入口先不实现。
- 不允许绕过人工审核直接公开发布；“自动发布”仅表示人工审核通过后由系统执行发布。
- 本阶段不承诺真实内容上线；真实来源、授权、投诉联系人和生产统计供应商仍需确认。

## 关键业务规则

1. 未发布、审核中、已驳回、已下架、投诉冻结内容不得出现在首页、频道、搜索、热榜、专题或收藏可读结果中。
2. 前端详情必须展示标题、分类标签、文章内容、作者、来源和发布时间；后台追溯字段不得作为普通前端信息展示。
3. 来源启用前必须配置授权类型、可展示字段、原文链接规则、投诉联系人和停用规则。
4. 自动审核必须输出风险等级和命中原因；所有公开发布都需要人工审核通过。
5. 人工审核通过后支持立即发布、定时发布和批量发布。
6. 定时发布前如内容被下架、投诉冻结、来源停用或关键字段修改，必须阻止发布并记录原因。
7. 投诉成立后，5 分钟内前台详情不可读，30 分钟内搜索、热榜、专题、推荐位、收藏打开结果不再展示可读内容。
8. 来源停用不是删除；后台仍需保留原始数据和审核/投诉/下架记录。

## 指标目标

| 指标 | 默认目标 | 验收窗口 |
| --- | --- | --- |
| 首页首屏主要内容可见 | P75 <= 2.5 秒 | 上线前固定环境验收，上线后连续 7 天复核 |
| 详情停留 | 中位停留 >= 25 秒，停留 >= 10 秒占比 >= 50% | 连续 14 天，详情 PV >= 1000 |
| 热榜点击率 | 热榜点击 / 热榜曝光 >= 8% | 连续 7 天，热榜曝光 >= 500 |
| 搜索有结果率 | 有结果有效查询 / 有效查询 >= 85% | 连续 7 天，有效搜索 >= 100 |
| 分享点击率 | 详情页分享点击 / 详情页访问 >= 1.5% | 连续 14 天，详情 PV >= 1000 |

流量不足时不以业务转化指标失败作为阻塞，但必须验收埋点完整性、口径正确性和后台可查看性。

## 待确认项

- 首批真实来源清单、授权材料、授权证明是否上传附件。
- 平台统一投诉联系人与来源专属联系人配置。
- 紧急责任人和投诉 SLA 是否可按默认口径执行。
- 生产统计/错误上报供应商、最终保留策略和用户告知机制。
- 微信登录入口恢复进入实现范围时的开放平台主体、回调域名、审核材料和账号绑定规则。
