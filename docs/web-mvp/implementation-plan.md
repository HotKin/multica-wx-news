# 快闻 Web MVP 工程实现计划

## 已完成首轮

KIN-16 已报告完成：

- Next.js 16 + React + TypeScript Web MVP 工程骨架。
- `dev/build/typecheck/lint/test/verify` 验证入口。
- 首页、频道、新闻详情、专题、搜索、后台骨架页。
- `/api/v1` 公开接口、`/api/admin/v1` 只读后台契约、`/healthz`、`/readyz`。
- 收藏接口边界，未登录返回 401。
- 领域类型、来源合规/审核快照 mock、埋点事件校验、单元测试。

当前仓库可见项目是微信小程序仓库；上述 Web 应用代码不在本仓库历史中。本次同步资料，不迁移不可见代码。

## 下一阶段工程补全

1. 数据层与迁移框架：Source、Article、Review、PublishJob、Complaint、Takedown、Topic、HotRank、AuditLog、Favorite、AnalyticsEvent。
2. RBAC 与审计：角色矩阵、越权失败、高风险操作理由、before/after、操作者与时间。
3. 内容状态机：抓取/录入、自动审核、人工审核、发布、定时/批量发布、驳回、下架、恢复、删除。
4. 投诉下架全链路：详情、搜索、热榜、专题、推荐位、收藏打开结果、缓存/索引/sitemap 清理。
5. 公开体验联调：首页、频道、详情、热榜、专题、搜索、分享、SEO、响应式。
6. 统计与埋点：白名单事件、敏感字段禁入、基础聚合。
7. 测试：状态机、权限、下架清理、审计、API、E2E 和响应式回归。

## 实现约束

- 可以使用 mock/seed 数据，但不得把未确认来源当作生产上线来源。
- 不接入生产第三方密钥或真实采集凭证。
- 不接入生产统计 SDK，除非业务/法务确认供应商和数据边界。
- 真实内容上线前必须完成安全复核和真实验收。
