# 快闻 Web MVP 安全与合规审查

## 结论

MVP 可以继续实现补全，但真实内容上线必须先把版权授权/来源合规、后台权限、隐私统计、下架删除、采集与搜索滥用控制落实为可验证能力。

## P0 阻断项

1. 未确认新闻源授权/转载边界前，禁止上线真实全文、图片、视频镜像和自动批量采集公开展示。
2. 未实现后台 RBAC、审核状态机和审计日志前，禁止上线运营后台写操作。
3. 未定义访问统计字段、日志脱敏、第三方 SDK 数据流和保留周期前，禁止接入生产统计与错误上报。
4. 未实现下架删除全链路清理前，禁止公开 SEO sitemap、站内搜索索引和热榜/专题/推荐闭环。
5. 未实现采集 allowlist、限速、熔断和禁用开关前，禁止长期自动化采集任务运行。
6. 未完成 XSS、open redirect、SSRF 基线测试前，禁止上线搜索、分享预览、外链图片代理或富文本展示。

## 必须落地的安全要求

| 范围 | 要求 |
| --- | --- |
| 来源合规 | 每个 source 必填授权类型、允许展示字段、原文链接规则、投诉联系人、启停规则和审计记录。 |
| 审核发布 | 自动审核不得绕过人工审核；未人工通过内容不可公开。 |
| 下架删除 | 下架/投诉必须清理详情、搜索、热榜、专题、推荐位、收藏打开结果、cache/CDN、sitemap。 |
| 后台权限 | RBAC、CSRF、IDOR 防护、高风险操作理由、before/after 审计和最小权限。 |
| 日志埋点 | 禁止记录明文密码、验证码、手机号、token、完整敏感 query、Authorization 和 source credential。 |
| 搜索分享 | 输入校验、限流、XSS 净化、分享参数白名单、open redirect 拦截。 |
| 采集任务 | allowlist、robots/TOS 或授权标记、频率限制、重试上限、熔断、禁用开关、SSRF 防护。 |
| 密钥 | 客户端 bundle 不得包含服务端密钥；第三方 SDK/API key 按环境隔离。 |

## 可接受风险

- 当前可继续 mock/沙箱实现，不接入真实第三方密钥和未授权来源。
- 基础匿名统计可接受，但仅采集 MVP 指标所需字段。
- 热榜和专题人工配置可接受，但必须审计、可回滚、可解释。

## 复验要求

安全复核需要在工程实现后覆盖：

- 权限矩阵和越权失败。
- 状态机非法流转。
- 投诉下架 SLA。
- XSS/open redirect/SSRF payload。
- 日志与埋点敏感字段抽样。
- 密钥扫描和客户端 bundle 检查。
